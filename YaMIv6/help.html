YaMI: yet another Modeller interface.

Fill the "PIR Sequence File" text box with the name of your PIR file, fill the
"Template PDB Codes" text box with valid PDB Codes for template structures,
these should be in lower case using the standar four-character name and separated
by spaces. A 5th character is valid too, to represente the chain to be modelled.

"PIR Sequence File", "Template PDB Codes" and "Number of Models" are required files.

Some Top file parameters
If "Number of Models" field is different than 1, then Modeller will be run with a top file with
'deviation' set to 4 ( the default value is 1 ). The maximum "Number of Models" is 5.

If the "Alignment file" is set to "No", the seg file required to run Modeller will be
automatically generated by the pdb2seg program on the server side.

If the "Alignment" field is set to "Yes", then "Alignment file" and "Target code" fields
are required fields too, and Modeller will be run with a top file with the 'routine' set
to "model" ( the default value is 'routine'="full_homol" ).
