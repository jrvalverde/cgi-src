  40  INSTALLATION
  41  ============
  42  
  43      QUICK & DIRTY
  44      -------------
  45  
  46  To install, simply uncompress and extract the distributed sources on any
  47  directory within your web server document area.
  48  
  49  First you need to create the database on MySQL. In order to do this,
  50  you may use the script provided under data/RB.tab.mysql : first you
  51  should edit this file and set an appropriate password for the database
  52  user, then source it from mysql.
  53  
  54  Secondly you must edit config.inc to set the password for the database
  55  user to the same you used in the RB.tab.mysql when creating the mysql
  56  database user.
  57  
  58  You need to initially populate the database with seed data. This may be
  59  obtained from the main web site:
  60  
  61      http://www.es.embnet.org/Services/MolBio/rbdb/.
  62  
  63  Simply make an empty query and save the results as TAB-delimited format.
  64  Then you may import the data on your local mySQL server using the provided
  65  forms and scripts.
  66  
  67  If you have any doubts, problems, etc.. please contact the author of the
  68  web interface:
  69  
  70      jr@acm.org

  73      DETAILS
  74      -------
  75      
  76      1. Creating the database
  77      
  78  The database is managed through MySQL. In future versions we may consider
  79  using ADODB so it becomes independent of the underlying engine, but for
  80  now you are stuck with using MySQL. It is free, it is GNU, it is terribly
  81  good... not that bad.
  82  
  83  First you must create the database, and associated table. Right now this is
  84  a very simple database: it only contains one table!
  85  
  86  The script ./data/RB.tab.mysql may help you get started creating the database,
  87  its table and loading an initial dataset in Tab-delimited format.
  88  
  89  This script will create a database called "rbdb". If you want to use another
  90  name, edit the script appropriately and note down the new name for later.
  91  
  92  Then, it will add a new user with privileges to maintain and query the
  93  database. This user is also called "rbdb". Again if you would rather have
  94  it called otherwise, just edit the script and note down the new name. Note
  95  that you MUST also change the name of the computers from which this user
  96  is going to access the database (i.e. the web server(s) that will be
  97  running these scripts).
  98  
  99  Finally, a password must be set for this user. YOU MUST ABSOLUTELY EDIT
 100  the script and use a good password for this user. If nothing else, this
 101  must absolutely be done: you may use the default database and user name
 102  but must change the password to something SAFE. Note it down as well.
 103  
 104  If you run the script from mysql, you will get a new user and database
 105  added, the later populated with the seed data you selected. Now you can
 106  move to the next step.
 107  
 108      2. Configuring the user interface
 109      
 110  Not much to be done here: simply edit the file config.inc and state on
 111  it the host where mysql is running, the name of the database, the user
 112  name to manage and access the database and the password for this user
 113  name.
 114  
 115  This file is protected by using a .htaccess file that has been crafted
 116  for Apache. YMMV and if you use a different Web server you will need to
 117  protect if from peeping eyes to avoid publishing your mysql user/password.
 118  
 119      3. You are done
